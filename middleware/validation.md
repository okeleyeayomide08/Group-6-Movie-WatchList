# Movie Validation

This explain the validation logic use in validation.js, This helped to check the movie objects before they're created or updated

## Purpose

Before we saved a movie to storage (movie.storage.js). it needs to be checked for correct structure, types, and values ranges. This module help us to handles that

## Expected Movie Schema

{
id: "unique-id",
title: "Bahubali",
genre: "Action",
releaseYear: 2016,
watched: true,
rating: 8,
createdAt: "2026-07-02T21:40:00.000z",
updatedAt: "2026-07-02T21:40:00.000Z",
}

## Rules (Field, Type, Required and Notes)

Title most be string type, it's required and must be non-empty (not just whitespace)

Genre is a string type, it's required and must match one of the VALID_GENRES we listed, ignoring case ("action", "ACTION", and "Action" all pass)
The stored value keeps whatever casing the client sent — validation does not normalize it

releaseYear is a number type, Must be a number (not a numeric string like "2016"), it's required and must be integer between 1888 and the current year

Watched is a boolean type, it's required and can only be true or false only

Rating is a number type, not required and if present ,since a movie may not be rated until it's been watched and the rating must be between 1 and 10

ID is a string type and this will be generated by the server, not client-supplied

createdAt is a string type and ISO timestamp generated by the server

updatedAt is a string type and follow same procedure as createdAT

## Functions

### validateMovie(movie, options)

Checks a movie object and returns whether it's valid, along with a list of any errors found.

Parameters

movie — the movie object to validate
options.isUpdate — boolean, defaults to false. When true, only the fields actually present on movie are checked (for partial updates via PATCH/PUT). When false, all required fields must be present (for POST/create).

Returns
{
isValid: true | false,
errors: ["array of readable error strings"]
}

All errors are collected in a single pass, so the caller sees every problem at once rather than one at a time

Example
import { validateMovie } from "./middleware/validation.js";

const result = validateMovie({
title: "Bahubali",
genre: "Action",
releaseYear: 2016,
watched: true,
rating: 5,
});

console.log(result);
// { isValid: true, errors: [] }

### validateMovieMiddleware(req,res,next)

Express Middleware wraps validateMovie for use directly in routes

#### list functions

1 Detects whether the request is a create or an update based on req.method (PATCH/PUT → update, otherwise → create)
2 Validates req.body against the schema
3 If invalid: responds immediately with 400 Bad Request and a JSON body of { errors: [...] } — the route handler is never reached
4 If valid: calls next() to continue to the actual route handler

Examples

import express from "express";
import { validateMovieMiddleware } from "../middleware/validation.js";

const router = express.Router();

router.post("/movies", validateMovieMiddleware, createMovieHandler);
router.patch("/movies/:id", validateMovieMiddleware, updateMovieHandler);

export default router;
